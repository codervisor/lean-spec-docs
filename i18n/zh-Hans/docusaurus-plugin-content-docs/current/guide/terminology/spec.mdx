# 规范

**规范**是一个轻量级文档，用于捕获工作单元的意图、约束和成功标准。它的编写方式使人类和 AI 代理都能理解接下来需要发生什么。

## 为什么重要
- 在交接和 AI 会话之间充当持久记忆
- 在实现开始之前强制明确问题
- 通过概述成功标准和护栏来支持自主工作

## LeanSpec 的结构
1. **前置元数据** — 机器可读的元数据（状态、优先级、标签）
2. **概述** — 问题、为什么重要以及期望结果
3. **计划** — 阶段、实现说明或验收标准
4. **测试/验证** — 如何确认工作已完成

将主 README 保持在约 300 行（≈2,000 令牌）以内，并在需要时将详细信息推送到子规范文件中。

## 快速示例
```markdown
# 042-api-rate-limiting
实现令牌桶速率限制（每个 API 密钥 100 请求/分钟）
以保护 API 免受 2% 重度用户的滥用。
```
