# LeanSpec 术语

新贡献者应该在深入了解各个术语页面之前从这里开始。下面的每个概念都链接到其深入页面，以便你可以直接跳转到所需的定义。

| 术语 | 为什么重要 |
| --- | --- |
| [规范](/docs/guide/terminology/spec) | 定义工作的最小单元以及它如何从想法演变为实现。 |
| [状态](/docs/guide/terminology/status) | 解释支持看板和报告的生命周期状态（`planned`、`in-progress`、`complete`）。 |
| [SDD 工作流程](/docs/guide/terminology/sdd-workflow) | 描述人类和 AI 代理遵循的规范驱动开发循环。 |
| [子规范](/docs/guide/terminology/sub-specs) | 显示何时拆分工作以保持在上下文经济阈值以下。 |
| [依赖关系](/docs/guide/terminology/dependencies) | 涵盖 `related` 与 `depends_on`，使协调保持清晰。 |
| [标签和优先级](/docs/guide/terminology/tags-priority) | 记录保持规范有序的轻量级规划信号。 |

> 需要全局视图？从[什么是 LeanSpec？](/docs/guide/what-is-leanspec)开始，然后返回这里澄清特定术语。
