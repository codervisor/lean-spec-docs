# 规范驱动开发（SDD）

SDD 是 LeanSpec 的操作节奏：在规范中捕获意图，让人类和 AI 从中执行，并保持规范同步。

## 五个循环阶段
1. **发现** — 在开始新工作之前搜索现有规范。
2. **计划** — 创建或更新规范，使意图明确无误。
3. **实现** — 人类和 AI 代理遵循规范进行构建。
4. **验证** — 运行测试和 `lean-spec validate` 以确保质量。
5. **更新** — 标记状态、经验教训和后续工作。

每当出现新信息时，循环就会重新开始。规范是活文档，而不是仪式性文书工作。

## 工作流程中的角色
- **作者**澄清问题和期望结果。
- **实施者**（人类或 AI）执行计划。
- **审阅者**在状态更改之前检查成功标准。

## 工具支持
- `lean-spec list`、`search` 和 `view` 用于发现。
- `lean-spec create` 和 `update` 用于规划/状态。
- `lean-spec validate` 和 MCP 服务器用于自动化。
