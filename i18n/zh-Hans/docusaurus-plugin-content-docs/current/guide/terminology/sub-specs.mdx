# 子规范

子规范是规范目录内的额外 Markdown 文件（例如 `DESIGN.md`、`TESTING.md`、`IMPLEMENTATION.md`）。它们让你保持主 README 简洁，同时仍然捕获深度。

## 何时创建
- README 超过约 300 行或 2,000 令牌。
- 详细内容（设计图、迁移步骤）会分散对概述的注意力。
- 不同受众需要不同深度（执行摘要与实现细节）。

## 常见模式
```
specs/082-web-sync/
  README.md          # 概述 + 决策上下文
  DESIGN.md          # 架构图和流程
  IMPLEMENTATION.md  # 阶段或任务列表
  TESTING.md         # 测试计划、覆盖策略
```

## 好处
- 通过分区详细信息来保持上下文经济。
- 让 AI 代理仅检索所需的文件。
- 鼓励渐进式披露——在感到痛苦时添加结构。
