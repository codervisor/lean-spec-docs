# 状态

规范的**状态**传达了工作在规范驱动开发（SDD）工作流程中的位置。

## 默认生命周期
1. **planned**（已计划）— 意图已捕获，实现尚未开始。
2. **in-progress**（进行中）— 工作正在积极进行中。
3. **blocked**（阻塞中）— 外部依赖阻止了进度。
4. **complete**（已完成）— 实现完成并验证。
5. **archived**（已归档）— 仅作为历史参考。

保持状态准确。在实现开始时立即更新它（而不是在编写规范时），在工作完成时再次更新。

## 为什么状态重要
- 使看板和时间线视图反映现实。
- 帮助 AI 代理决定哪些规范可以安全修改。
- 无需会议即可让团队即时可见。

## 前置元数据示例
```yaml
---
status: in-progress
created: 2025-11-16
priority: high
tags: [api, backend]
---
```
