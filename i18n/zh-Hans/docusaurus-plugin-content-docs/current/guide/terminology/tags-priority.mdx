# 标签和优先级

标签和优先级值使规范可过滤，并帮助团队集中注意力。

## 标签
添加描述性标签，帮助你快速切分工作。
```yaml
tags: [api, backend, security]
```
- 使用一致的词汇（`frontend`、`backend`、`docs`）。
- 每个规范保持 2-4 个标签以保持清晰。
- 与 `lean-spec list --tag <tag>` 结合使用以突出显示相关规范。

## 优先级
表达紧迫性或业务影响。
```yaml
priority: high
```
典型级别：`low`、`medium`、`high`、`critical`。当现实发生变化时更新优先级——旧值会导致规划债务。

## 为什么重要
- 驱动看板视图、仪表板和 AI 分类提示。
- 帮助 AI 代理选择下一个最有价值的规范。
- 提供轻量级对齐，无需单独的电子表格。
