# 什么是 LeanSpec？

LeanSpec 是一种为 AI 时代设计的轻量级 **规格驱动开发（Spec-Driven Development, SDD）** 方法论。它帮助团队——包括人类和 AI 代理——专注于 *意图* 而非实现细节。

> **可以将其视为"项目管理的 Markdown"**：简单、结构化、机器可读。

## 你将获得什么

*   [**核心 CLI**](/docs/guide/usage/cli/creating-managing)：管理规格说明、跟踪状态并可视化依赖关系（`lean-spec board`、`lean-spec deps`）。
*   [**MCP 服务器**](/docs/guide/usage/mcp-integration)：让 AI 代理（Claude、Cursor 等）直接访问和搜索你的规格说明。
*   [**可视化模式**](/docs/guide/visual-mode)：用于浏览项目规格说明图的精美 Web 界面（`lean-spec ui`）。

## 一个简单示例

LeanSpec 只是一个带有前置元数据的 Markdown 文件：

```markdown title="./specs/001-user-auth-api/README.md"
---
status: in-progress
created: 2025-11-07
priority: high
tags: [api, auth]
---

# 用户认证 API

## 目标
为移动应用启用安全的用户登录和会话管理。

## 关键场景
1. 用户使用邮箱/密码登录 → 接收 JWT 令牌
2. 用户使用令牌访问受保护端点 → 获取数据
3. 用户令牌过期 → 接收 401，必须重新认证

## 验收标准
- [ ] 登录端点在成功时返回有效的 JWT
- [ ] JWT 包含用户 ID 和过期时间
- [ ] 受保护端点验证 JWT 签名

## 非目标
- 社交登录（未来功能）
- 密码重置（单独的规格说明）
```

你可以使用你喜欢的 AI 编码工具来读取此规格说明，并基于它生成代码、文档甚至测试用例。
